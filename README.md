# Sprint-Review-and-Retrospective
This work pulls together the Agile and Scrum practicesn applied in Chada tech company.

User needs are requests and features that users want to see in the software delivered in demos after every Sprint. Creating user stories from user needs helps this process because it is the basis for starting any implementation of modifications or new implementations in the software. User Stories also help to make the whole team understand the requirements, so the production and testing can achieve more beneficial results. 

I use Agile methodology in my projects; this approach allows me to make incremental progress, so I can have Demos after every Sprint that I propose to myself. Personally, I don't like to let the project get too many changes before a new commit, it is always better for the project to test and build by stories. 

To be a good team member in software development is to understand your role, every role has different tasks that have to be done, implementing the Done meaning for Agile projects, so the project can succeed. Always helping the team members and reminding them that there are roles in projects to help you in any situations is presented during development, and communication is the most important part of Agile; a project succeeds depending on the good communication of the team.
